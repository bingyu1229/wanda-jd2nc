# 科目表生成：HTM → CSV

## 1. 将 HTM 转为 CSV

- 使用脚本：`test/step_A.ts`
- 在项目根目录执行（需已 `npm install`）：

```bash
npx tsx test/step_A.ts
```

- 默认输入：从当前执行目录开始，递归查找所有子目录中的固定文件名 `科目.htm`；如需调试单个文件，仍可显式传入输入路径。
- 默认输出：每个 `科目.htm` 所在目录内生成 `a.csv` 和 `科目导入.csv`，生成文件保留在原子目录。
- 转换时会 **跳过 HTML 表内第 1 行**（`第1列`…`第18列` 占位表头），**删除第 3 列（助记码）**；CSV 首行为 `科目代码`、`科目名称`、… 等业务表头。
- 可选参数：
  - `--encoding gb18030`：输入编码（默认 gb18030，兼容 gb2312）
  - `--utf8-bom`：输出带 UTF-8 BOM，便于 Windows 下 Excel 打开
  - `--excel-text-cols 0,1`：指定列以 Excel 文本公式形式写出
  - `--no-normalize-col-a`：跳过 **第 2 节** 列 A 的全部处理（2.1–2.4），列 A 保持 HTM 原样

说明：转换阶段不按数字解析单元格，避免日期/期间等文本被截断或丢末尾 `0`。

## 2. 对列 A（科目代码）规范化（同一操作对象）

仅对「整格为英文字母、数字与点号组成」的科目代码做改写；**表头、中文、其他非代码文本**等保持原样。  
科目表中的 `1911.01`、`2002.01` 等仍按科目代码处理；若处理对象不是科目表、列 A 可能是期间或日期，请使用 `--no-normalize-col-a` 跳过列 A 规范化。

### 2.1 在表最后新增一列，复制原本的列 A（科目代码），列名为`原科目代码`

### 2.2 规则摘要：`4.(3)*`（带点）

- **第一段**（第一个 `.` 之前）：固定为 **4 位字符**，不足时在 **右侧** 用 `0` 补齐；超过 4 位时取前 4 位，**剩余字符拼接到下一段**（与下一段原内容首尾相接后再切分）。
- **每一段**（第一个 `.` 之后的每一个「点与点之间」的片段）：在该片段内部从左到右每 **3** 个字符一组；该片段最后一次不足 3 个字符时在 **右侧** 用 `0` 补满 3 个字符。英文字母按字符参与分组，规则与数字相同。  
**不会** 把多个片段的字符先拼成一条再按 3 位切（否则 `1151.13.01` 会错误变成 `1151.130.100`）。
- **仅一段且无后续层级时**：若只有第一段 4 位、后面没有任何 3 位组，则 **只保留这 4 位**，不再追加 `.000` 等。

### 2.3 带点形态示例（2.1 的结果，仍含 `.`）


| 原始 A 列            | 带点结果                   |
| ----------------- | ---------------------- |
| `1001`            | `1001`                 |
| `123`             | `1230`                 |
| `1001.001`        | `1001.001`             |
| `1002.01`         | `1002.010`             |
| `1151.13.01`      | `1151.130.010`         |
| `123.34.456.3456` | `1230.340.456.345.600` |
| `2131.003.HA0302` | `2131.003.HA0.302`     |
| `2131.003.HA11`   | `2131.003.HA1.100`     |


### 2.4 改造重复 `科目代码`

在 2.2–2.3 得到带点串之后、2.5 去掉 `.` 之前，确保科目代码唯一：

- 第一次出现的相同代码，将最后一位数字变为`a`；
- 第二次出现的相同代码，将最后一位数字变为`b`；
- 第三次出现的相同代码，将最后一位数字变为`c`；
- 以此类推；
- 第27次出现的相同代码，将最后两位数字变为`aa`；
- 以此类推；如果追加后的代码已被占用，则继续递增直到唯一。

示例：


| 2.2 带点结果   | 2.3 去重后    |
| ---------- | ---------- |
| `1234.100` | `1234.10a` |
| `1234.100` | `1234.10b` |
| `1234.100` | `1234.10c` |


### 2.5 去掉小数点（最终写入 CSV 的列 A）

在得到 **2.1–2.2 的带点串** 之后，**删除所有 `.`**，得到连续代码串（便于部分导入或匹配场景）。

示例：`1234.567.890` → `1234567890`。


| 原始 A 列            | 最终 A 列（无 `.`）      |
| ----------------- | ------------------ |
| `1001`            | `1001`             |
| `123`             | `1230`             |
| `1001.001`        | `1001001`          |
| `1002.01`         | `1002010`          |
| `1151.13.01`      | `1151130010`       |
| `123.34.456.3456` | `1230340456345600` |
| `2131.003.HA0302` | `2131003HA0302`    |
| `2131.003.HA11`   | `2131003HA1100`    |


（先按 2.2 得到带点串，再按 2.5 去掉全部 `.`。）

## 3. 改造重复 `科目名称`

在科目代码完成规范化与去重之后，检查 `科目名称` 是否重复。若同一个科目名称出现多次，则每条重复记录的 `科目名称` 后追加该行唯一的 `科目代码`。

示例：


| 科目代码      | 原科目名称 | 改造后科目名称      |
| --------- | ----- | ------------ |
| `1234567` | `人民币` | `人民币1234567` |
| `1234789` | `人民币` | `人民币1234789` |


## 4. 生成「科目导入.csv」

运行 `a.ts` 生成第 2 步 CSV 时，会在输出 CSV 的同一目录额外生成：

- `科目导入.csv`

生成规则：

1. `科目导入.csv` 的 A1 固定写入：
  ```text
   bd_accsubj_$head,subjcode,subjname,pk_subjtype,balanorient,period,outflag
  ```
2. 第 2 步生成 CSV 的 A、B、C、D 列复制到 `科目导入.csv` 的 B、C、D、E 列。
3. `科目导入.csv` 的 A 列除 A1 外全部留空。
4. F 列表头为 `有效期`，每条数据行固定写入 `2000-01`，行数与第 2 步 CSV 的记录数保持一致。
